# Role Permissions

The **Role Permissions** module is used to define roles and control which permissions belong to each role.

![Role Permissions](/img/hr/ahr/role-permissions.png)

## Role List

The left panel contains the role list. Each row shows:

- **Name**
- **Created On**
- **Actions** for edit and delete

Use the **New** button to create a new role.

## Create Role

Click **New** to open the role creation dialog.

![Create Role](/img/hr/ahr/role-permissions-add.png)

Enter the **Role Name** and click **Submit** to save it.

## Edit Role

Use the **Edit** icon in the role list to update an existing role name.

## Delete Role

Use the **Delete** icon in the role list to remove a role.

![Delete Role](/img/hr/ahr/role-list-delete.png)

You will be asked to confirm the deletion before the role is removed.

## Assign Permissions

Click **Assign Permission** in the right panel to manage the permissions attached to the selected role.

![Assign Permission](/img/hr/ahr/role-permissions-add-assign-permission.png)

The permission dialog supports:

- Expanding grouped permissions
- Selecting individual permissions with checkboxes
- Using **Select All** and **Deselect All**

![Select Permissions](/img/hr/ahr/role-permissions-add-assign-permission-select.png)

After choosing the required permissions, click **Submit** to save the assignment.

:::note
Permissions are grouped by module. Assign only the minimum access needed for the role so users inherit the correct menu visibility and actions.
:::
