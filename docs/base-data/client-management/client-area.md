# Client Area

The `Client Area` module is used to maintain area or region names for clients. These areas are used in the client form so users can assign each client to the correct geographic or business area.

## Client Area List

![Client Area List](/img/basedata/client-management/client-area/client-area-list.png)

Go to `Client Management > Client Area` to view all area records.

### Information Shown on This Page

- **Active**: Shows whether the area is currently active.
- **ID**: System-generated area ID.
- **Area Name**: Name of the client area.
- **Created On**: Date when the area was created.
- **Actions**: Options to edit or delete the area.

### Top Controls

- **Search**: Finds an area quickly by name or ID.
- **New**: Opens the area entry form.
- **All**: Displays all areas.
- **Active**: Displays only active areas.
- **Inactive**: Displays only inactive areas.

## Add a New Client Area

![Client Area Add](/img/basedata/client-management/client-area/client-area-add.png)

Follow these steps to create a new client area:

1. Go to `Client Management > Client Area`.
2. Click **New**.
3. Turn on the **Active** toggle if the area should be available for use immediately.
4. Enter the name in **Area Name**.
5. Click **Submit**.

### Field Description

- **Active**: Keeps the area available in the system.
- **Area Name**: Enter the client area name. This field is required.

## Edit a Client Area

![Client Area Edit](/img/basedata/client-management/client-area/client-area-edit.png)

Follow these steps to update a client area:

1. Find the area in the list.
2. Click the **Edit** icon.
3. Update the **Active** status or **Area Name**.
4. Click **Submit** to save the changes.

## Delete a Client Area

![Client Area Delete](/img/basedata/client-management/client-area/client-area-delete.png)

Follow these steps to delete a client area:

1. Locate the area in the list.
2. Click the **Delete** icon.
3. In the warning dialog, click **Delete** to confirm.
4. Click **Cancel** if you do not want to continue.

:::warning
Do not delete a client area if it is already assigned to active clients. Mark it inactive if you want to stop using it without removing historical references.
:::
